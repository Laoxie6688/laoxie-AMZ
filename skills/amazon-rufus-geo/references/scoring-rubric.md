# Rufus GEO 六维评分标准（100 分制）

> **版本说明**：本评分标准已按 SKILL.md 2026-04 版更新。如果 scoring-rubric.md 和 SKILL.md 的权重不一致，以 SKILL.md 为准。

---

## 权重总览

| 维度 | 分值 | 变化 |
|------|------|------|
| ① 标题 | 15 分 | ⬇️ 从 20 降至 15 |
| ② 五点描述 | 20 分 | ⬇️ 从 25 降至 20 |
| ③ 产品描述/A+ 内容 | 20 分 | ⬆️ 从 15 升至 20 |
| ④ 后台搜索词 | — | ❌ 已移除（Agent 无法获取此数据）|
| ⑤ Q&A 策略 | 25 分 | ⬆️ 从 15 涨至 25 |
| ⑥ 数据一致性 | 10 分 | — 不变 |
| ⑦ 图片/Alt-text | 10 分 | ⬆️ 从 5 升至 10（后台搜索词分值转入）|
| **合计** | **100 分** | |

**权重调整逻辑**：
- Q&A 权重最高（25 分）：Rufus 把 UGC（评论 + Q&A）当作 Ground Truth，权重高于品牌自述
- 后台搜索词已移除：Agent 无法获取此数据，分值转入产品描述（+5）和图片/Alt-text（+5）
- 新增图片/Alt-text（5 分）：Rufus 官方确认会"add descriptive tags to product images"，图片成为独立信号
- 标题和五点相对降权：不是不重要，是 Q&A 和 UGC 的相对重要性上升了

## ① 标题（15 分）

| 分数区间 | 标准 |
|----------|------|
| 13-15 分 | 核心关键词 + 主要使用场景 + 自然语言；前 80 字符完整表达核心购买理由；150-200 字符；至少覆盖 3 种 COSMO 关系 |
| 9-12 分 | 有核心关键词但缺场景描述；长度合理但前 80 字符不够有力 |
| 5-8 分 | 关键词不全或结构混乱；缺少场景和人群信息 |
| 0-4 分 | 严重问题（见下方致命错误） |

**致命错误（直接扣 12 分）**：
- "Gift for Dad Men Him Husband Boyfriend" 式关键词堆砌
- Rufus LLM 判定为垃圾内容，直接降低整个 Listing 信任分

**检查要点**：
- [ ] 前 80 字符包含最核心购买理由 + 至少 1 个使用场景
- [ ] 读起来像人说的话，不像关键词列表
- [ ] 至少自然覆盖 3 种 COSMO 关系

---

## ② 五点描述（20 分）

| 分数区间 | 标准 |
|----------|------|
| 17-20 分 | 每条回答一个真实买家问题；15 种 COSMO 关系覆盖 ≥ 10 种；使用「场景痛点 → 独特机制 → 技术支撑」公式；自然语言 |
| 12-16 分 | 大部分用自然语言但 COSMO 覆盖率 < 10 种；场景描述不够具体 |
| 6-11 分 | 功能导向而非场景导向；覆盖率 < 7 种 |
| 0-5 分 | 严重问题（见下方致命错误） |

**致命错误（该条直接 0 分）**：
- 纯规格堆砌（如 "IP68, 6000mAh, 30L, 2.1lbs"）
- 关键词堆砌式写法

**每条 bullet 检查要点**：
- [ ] 是否回答了一个真实的买家问题？
- [ ] 是否使用「场景痛点 → 机制 → 技术」结构？
- [ ] 该条自然覆盖了哪 2-3 种 COSMO 关系？（标注出来）
- [ ] 长度在 200-500 字符之间？

**五条建议覆盖维度**：
1. 核心功能 + 最大卖点（USED_FOR_FUNC + CAPABLE_OF）
2. 材质/品质 + 耐久性（CAPABLE_OF + USED_BY 社会证明）
3. 使用场景 + 目标人群（USED_FOR_EVE + USED_FOR_AUD + USED_ON/USED_IN_LOC）
4. 差异化 + 互补搭配（USED_AS + USED_WITH）
5. 包装内容 + 售后保障（XWANT 风险消除 + XIS_A 用户自我定位）

## ③ 产品描述/A+ 内容（20 分）⬆️

| 分数区间 | 标准 |
|----------|------|
| 17-20 分 | 覆盖多种使用场景和用户人群；有对比信息辅助决策；扩展覆盖了五点没提到的 COSMO 关系；A+ 图片 alt-text 全部完整 |
| 12-16 分 | 内容丰富但场景覆盖不全；部分 alt-text 缺失 |
| 6-11 分 | 内容较少或纯文字堆砌；缺乏场景化描述 |
| 0-5 分 | 无 A+ 内容或 A+ 仅为图片无文案 |

**检查要点**：
- [ ] 是否覆盖了五点没提到的 COSMO 关系？
- [ ] 是否有对比信息帮助买家做决策？
- [ ] 是否有使用视频？
- [ ] A+ 图片 alt-text 是否全部完整填写？
- [ ] alt-text 描述的是"图片里发生了什么"而不是"图片是什么"？

## ④ 后台搜索词（已移除）

| 分数区间 | 标准 |
|----------|------|
此维度已移除。Agent 无法自动获取卖家后台搜索词，无法评分。
分值已重新分配：产品描述/A+ +5分（共20分），图片/Alt-text +5分（共10分）。

如用户提供后台搜索词，可作为加分参考，但不计入正式评分。

**检查要点**：
- [ ] 是否填满 250 字节？
- [ ] 是否避免了重复标题中已有的词？
- [ ] 是否包含长尾名词短语而非单字？
- [ ] 是否包含常见拼写变体和同义词？

**注意**：Rufus 不做关键词匹配，这个字段的作用更多是补充传统 A9/A10 搜索兼容性，而非直接影响 Rufus 推荐。

## ⑤ Q&A 策略（25 分）⬆️ 权重最高

| 分数区间 | 标准 |
|----------|------|
| 21-25 分 | Q&A ≥ 15 条；覆盖 ≥ 10 种 COSMO 关系；品牌方主动回答关键问题；回答详细自然含场景信息；差评高频问题已预防式回答 |
| 15-20 分 | Q&A 10-14 条；覆盖 7-9 种关系；回答质量良好 |
| 8-14 分 | Q&A 5-9 条；覆盖 < 7 种关系；回答简短或营销腔重 |
| 0-7 分 | Q&A < 5 条或无品牌方主动回答 |

**为什么权重最高**：
Rufus 的训练数据和 RAG 检索源明确包含 customer Q&As，且 Rufus LLM 对 UGC 的信任度**高于**品牌自述。Q&A 是卖家能"主动控制"的最高权重 UGC 信号。

**Rufus 特别偏好的 Q&A 类型（按召回频率排序）**：
1. `Can this [action]?` → CAPABLE_OF
2. `Is this good for [people/situation]?` → USED_FOR_AUD / XIS_A
3. `How long does it last / how durable?` → CAPABLE_OF + USED_BY
4. `What comes with it / what do I need?` → USED_WITH
5. `Is this waterproof / machine-washable?` → CAPABLE_OF

**检查要点**：
- [ ] Q&A 数量 ≥ 10 条？（≥ 15 条满分）
- [ ] 覆盖了多少种 COSMO 关系？
- [ ] 品牌方是否主动回答了关键问题？
- [ ] 回答是否详细、自然、包含场景信息？
- [ ] 评论里高频出现的问题有没有预先回答？
- [ ] 差评中的常见投诉有没有用预防式 Q&A 正面回应？

## ⑥ 数据一致性（10 分）

| 分数区间 | 标准 |
|----------|------|
| 9-10 分 | 标题/五点/描述/后台属性/主图 100% 一致；所有结构化字段填全填准 |
| 6-8 分 | 有 1 处小不一致但非致命（如颜色名称细微差异） |
| 3-5 分 | 有 2+ 处不一致，但非核心属性 |
| 0-2 分 | 存在致命错误（见下方） |

**致命错误**：
- 标题说 "3 Pack" 但后台 Item Package Quantity 填 "1" → **扣 8 分并标红**
- 标题材质 ≠ 后台 Material Type → **扣 6 分**
- 主图款式 ≠ 文案描述颜色 → **扣 5 分**

**Rufus 行为**：检测到数据冲突会**直接压制展示**——不是降权，是从对话推荐里剔除该 ASIN。因为 Rufus 的优先级是"不给用户糟糕体验"。

**建议**：数据一致性分数应自查到 9 分以上。低于 9 分要全面排查所有字段。

**检查要点**：
- [ ] 标题中的数量/材质/尺寸和后台属性是否完全一致？
- [ ] 主图展示的颜色/款式和文案描述是否一致？
- [ ] 后台 Occasion / Season / Target Audience 是否和文案呼应？
- [ ] 后台 Browse Node / Item Type 和实际品类是否匹配？

## ⑦ 图片/Alt-text（10 分）⬆️🆕

| 分数区间 | 标准 |
|----------|------|
| 9-10 分 | 主图白底清晰合规；≥ 2 张生活场景图；A+ 所有图片 alt-text 完整且描述性准确；有使用视频 |
| 6-8 分 | 主图合规；有场景图但 < 2 张；部分 alt-text 缺失 |
| 3-5 分 | 主图合规但无场景图；alt-text 大量缺失或无意义 |
| 0-2 分 | 主图不合规或完全无 alt-text |

**为什么新增这个维度**：
Rufus 官方能力确认："adds descriptive tags to product images"。Rufus 会读取图片并生成视觉标签，alt-text 是卖家给 Rufus 的图片说明书。

**Alt-text 写法要求**：
- ✅ 描述"图片里发生了什么"：`A woman adjusting the backpack's shoulder straps on a mountain trail at sunrise`
- ❌ 描述"图片是什么"：`Product image 3` 或 `Backpack photo`

**检查要点**：
- [ ] 主图是否白底清晰，符合 Amazon 主图规范？
- [ ] 是否有至少 2 张生活场景图（产品在使用环境中）？
- [ ] A+ 内容图片 alt-text 是否全部完整填写？
- [ ] alt-text 是否描述场景而非仅命名？
- [ ] 是否有使用视频？

## 评级标准

| 总分 | 评级 | 说明 |
|------|------|------|
| 90-100 | 🟢 优秀 | Rufus 高频推荐候选 |
| 75-89 | 🟡 良好 | 基础覆盖到位，有优化空间 |
| 60-74 | 🟠 及格 | 系统性问题需要优化 |
| 40-59 | 🔴 不及格 | 严重缺失，需要全面改写 |
| 0-39 | ⚫ 危险 | Rufus 可能主动压制展示 |

## 使用说明

1. **审计时**：逐维度打分，每维度给出分数 + 一句话评价 + 具体证据
2. **优化时**：从分数最低的维度开始改，优先修复致命错误
3. **如果 SKILL.md 和本文件权重不一致**：以 SKILL.md 为准
4. **致命错误不可用其他维度分数弥补**：一个致命错误直接拉低总分上限
